## Context Map

### Slice intent

This slice defines the execution-prep packet for the selected fallback path:

`governed baseline reset via parity rerun under frozen spec ri_p1_off_parity_v1`

It replaces evidence recovery as the intended next path.

It does **not** execute the rerun, does **not** grant baseline approval, and does **not** grant execution approval.

### Decision rationale

The repository already has:

- a frozen canonical artifact contract
- a named metadata contract
- a named future gate bundle
- a formal execution-plan packet
- a formal execution-review packet

The remaining uncertainty was baseline provenance recovery from old local / ignored / partially retained evidence.

The user has explicitly chosen to stop pursuing evidence recovery and instead use the governed fallback route.

This prep slice therefore records the selected future baseline path as:

- `newly approved baseline under explicit governance approval`

That is an allowed **future** baseline classification path only. This prep slice does not itself grant that approval and does not authorize execution.
