# Refactor candidates audit index

This README is a navigation aid for the files currently present in this folder.
It does not replace or reinterpret the underlying audit or evidence documents.
For authoritative details, open the individual candidate packet, review, or signoff files directly.

## What lives here

This folder mainly preserves historical candidate records from cleanup/refactor review work.
The observed file pattern includes:

- `command_packet_candidate*` — candidate-specific scope and intended action
- `*_signoff_*` — closure/signoff notes for a specific candidate subset
- `*_review_*` — review notes captured during candidate assessment

## Suggested reading order

1. Open the relevant `command_packet_candidate*` file for the candidate number you are investigating.
2. If present, read the matching `*_review_*` file next.
3. If present, read the matching `*_signoff_*` file last to understand recorded closure for that subset.

## Interpretation note

Treat this folder as a traceability archive for candidate decisions.
Do not infer current approval, deletion, or retention status beyond what each individual document explicitly records.
