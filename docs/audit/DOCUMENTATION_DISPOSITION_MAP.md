## Purpose

This audit answers one bounded question only:

- which currently reviewed Genesis-Core documentation surfaces appear safest to keep as-is
- which appear to need clearer status-marking or reclassification later
- which appear to have explicit future archive/supersession follow-up value
- which remain too ambiguous to treat as cleanup-ready

This file may be used as a planning and review aid for later docs-only cleanup slices.
It may **not** be used as cleanup authority by itself.
In this audit, `KEEP` means retain the document under its current framing only; it does not endorse substantive claims, confer approval, or establish current authority.

## Scope and method

This slice reviews only a bounded set of documentation surfaces that are currently likely to influence reader interpretation because they are:

- top-level routing or taxonomy guides
- derivative knowledge-layer surfaces
- branch/historical provenance documents with explicit current-use notes
- selected archived or plan documents whose location or wording may still invite active-status misreads

Method for this slice:

- read current branch/worktree text directly for each listed document
- prefer direct evidence from the document itself
- use current derivative routing/index surfaces only as corroborating evidence, not as replacement authority
- default to `UNKNOWN_KEEP` when evidence is incomplete, conflicting, indirect, or inferential

No dedicated repository skill was identified for this exact audit shape; this is a manual governed docs-only slice.

## Confidence rule

- `HIGH` â€” explicit role/status/supersession evidence exists in the reviewed file or directly cited companion file
- `MEDIUM` â€” multiple corroborating signals exist, but the follow-up is still partly judgmental
- `LOW` â€” evidence is partial or mostly inferential

If confidence would otherwise be `LOW`, the expected safe disposition is usually `UNKNOWN_KEEP`, `KEEP`, or at most `ADD_STATUS_HEADER`.

## Ambiguity rule

When evidence is incomplete, conflicting, indirect, or inferential, the required disposition is `UNKNOWN_KEEP`.
