## Exact research question

The future execution slice must answer only this bounded question:

### D1 context-clean selectivity question

On the already-materialized four-surface D1 bank below, and using only the admitted non-age family,
does at least one admitted field yield context-clean target-vs-context selectivity when reread as a fixed target-bank ceiling?

For the fixed bank $B$ and admitted field $f$, define:

$$
\text{target\_ceiling}(B, f) = \max\{f(x) : x \in \text{fixed claim-eligible target rows in } B\}
$$

and the fixed context-floor check:

$$
\text{context\_min}(B, f) = \min\{f(x) : x \in \text{fixed context rows in } B\}
$$

A field is a bounded context-clean survivor only if all of the following hold on the fixed four-surface bank:

1. the field is directly evaluable on every fixed target and every fixed context row required by this slice,
2. every fixed claim-eligible target row satisfies $f(x) \le \text{target\_ceiling}(B, f)$,
3. every fixed context row satisfies $f(x) > \text{target\_ceiling}(B, f)$,
4. equivalently, $\text{target\_ceiling}(B, f) < \text{context\_min}(B, f)$,
5. only the fixed target rows and fixed context rows listed below participate in PASS/FAIL adjudication,
6. no fresh threshold ranking, no subgroup retry, and no subject widening is used.

PASS is descriptive only and means at least one admitted non-age field satisfies the bounded context-clean survivor rule above.

FAIL is descriptive only and means every admitted field either:

- overlaps the fixed context bank,
- loses direct evaluability,
- or requires forbidden widening to say anything stronger.

This slice may conclude only one of the following:

- **bounded context-clean selectivity present** — at least one admitted non-age field produces a fixed target-bank ceiling that selects all fixed targets and excludes all fixed contexts
- **bounded context-clean selectivity null** — every admitted non-age field still leaks into the fixed context bank on the current exact four-surface reread

This slice may **not**:

- search a ranked threshold grid
- search a conjunction
- re-weight surfaces after seeing the data
- promote a context-clean result into harmful-vs-correct suppression authority
- reopen recurrence widening, transport logic, or runtime integration
